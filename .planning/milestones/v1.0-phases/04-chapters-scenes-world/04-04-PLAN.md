check_magic_compliance logic (WRLD-08 spec from CONTEXT.md):
- Inputs: character_id: int, magic_element_id: int, action_description: str
- Query 1: SELECT * FROM magic_system_elements WHERE id = ? → return NotFoundResponse if miss
- Query 2: SELECT * FROM practitioner_abilities WHERE character_id = ? AND magic_element_id = ?
- character_has_ability = True if ability_rows else False (empty list = not registered)
- applicable_rules: collect non-None values from element["rules"], element["limitations"], element["costs"]
- violations: if character_has_ability is False → append f"Character {character_id} has no registered ability for element {magic_element_id}"
- compliant = len(violations) == 0 and character_has_ability is True
- Does NOT write to magic_use_log — that is log_magic_use's job

Seed data for test assertions:
- magic_element_id=1 → name="Ember-Binding", rules/limitations/costs all populated
- practitioner_abilities: 1 row, character_id=1, magic_element_id=1, proficiency_level=3
- character_id=1 + magic_element_id=1 → has ability (character_has_ability=True)
- character_id=2 + magic_element_id=1 → no ability (character_has_ability=False)
- magic_use_log: 1 existing row, chapter_id=1, scene_id=1, character_id=1, element_id=1

    - First verify character exists: `SELECT id FROM characters WHERE id = ?`; return `NotFoundResponse(not_found_message=f"Character {character_id} not found")` if miss
    - Then: `SELECT * FROM practitioner_abilities WHERE character_id = ? ORDER BY id`
    - Return `[PractitionerAbility(**dict(row)) for row in rows]` (empty list is valid)

    **Tool 3 — log_magic_use (WRLD-07):**
    - Signature: `async def log_magic_use(chapter_id: int, character_id: int, action_description: str, magic_element_id: int | None = None, scene_id: int | None = None, cost_paid: str | None = None, compliance_status: str = "compliant", notes: str | None = None) -> MagicUseLog | NotFoundResponse | ValidationFailure`
    - Verify chapter exists first: `SELECT id FROM chapters WHERE id = ?`; return `NotFoundResponse` if miss
    - Append-only INSERT (no conflict resolution — magic_use_log has no UNIQUE constraint):
      ```sql
      INSERT INTO magic_use_log
        (chapter_id, scene_id, character_id, magic_element_id,
         action_description, cost_paid, compliance_status, notes, created_at)
      VALUES (?, ?, ?, ?, ?, ?, ?, ?, datetime('now'))
      ```
    - `new_id = cursor.lastrowid`, await conn.commit()
    - SELECT back: `SELECT * FROM magic_use_log WHERE id = ?`, return `MagicUseLog(**dict(row[0]))`
    - Wrap in try/except; return `ValidationFailure(is_valid=False, errors=[str(exc)])`

    **Tool 4 — check_magic_compliance (WRLD-08):**
    - Signature: `async def check_magic_compliance(character_id: int, magic_element_id: int, action_description: str) -> MagicComplianceResult | NotFoundResponse`
    - READ-ONLY — no DB writes
    - Two queries:
      1. `SELECT * FROM magic_system_elements WHERE id = ?` → return `NotFoundResponse` if miss
      2. `SELECT * FROM practitioner_abilities WHERE character_id = ? AND magic_element_id = ?`
    - `element = dict(element_rows[0])`

    - `compliant = len(violations) == 0 and character_has_ability`
    - Return `MagicComplianceResult(compliant=compliant, violations=violations, applicable_rules=applicable_rules, character_has_ability=character_has_ability)`

    - NO `print()` anywhere — `logger.error(...)` in except blocks
    - check_magic_compliance does NOT call conn.commit() — read-only tool
    - Return error types — NEVER raise
