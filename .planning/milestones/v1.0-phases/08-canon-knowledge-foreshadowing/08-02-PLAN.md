1. `get_reader_state(chapter_id: int) -> list[ReaderInformationState] | GateViolation`
   - LOCKED DECISION: cumulative semantics — `WHERE chapter_id <= ?` NOT `= ?`
   - SQL: `SELECT * FROM reader_information_states WHERE chapter_id <= ? ORDER BY chapter_id ASC`
   - Returns list (may be empty)

2. `get_dramatic_irony_inventory(include_resolved: bool = False, chapter_id: int | None = None) -> list[DramaticIronyEntry] | GateViolation`
   - LOCKED DECISION: unresolved only by default
   - Build WHERE clause dynamically:
     - if not include_resolved: add `resolved = FALSE`
     - if chapter_id is not None: add `chapter_id = ?`
   - SQL: `SELECT * FROM dramatic_irony_inventory` + WHERE clause + `ORDER BY created_at ASC`
   - Returns list (may be empty)

3. `get_reader_reveals(chapter_id: int | None = None) -> list[ReaderReveal] | GateViolation`
   - Optional chapter_id filter: if provided, `WHERE chapter_id = ?`; else no filter
   - SQL: `SELECT * FROM reader_reveals` + optional WHERE + `ORDER BY created_at ASC`
   - Returns list (may be empty)

4. `upsert_reader_state(chapter_id: int, domain: str, information: str, reader_state_id: int | None = None, revealed_how: str | None = None, notes: str | None = None) -> ReaderInformationState | GateViolation`
   - IMPORTANT: reader_information_states has UNIQUE(chapter_id, domain) — both branches use ON CONFLICT
   - None-id branch: INSERT with `ON CONFLICT(chapter_id, domain) DO UPDATE SET information=excluded.information, revealed_how=excluded.revealed_how, notes=excluded.notes` — use cursor.lastrowid (returns existing id on conflict — correct SQLite behavior)
   - Provided-id branch: INSERT with `ON CONFLICT(id) DO UPDATE SET information=excluded.information, revealed_how=excluded.revealed_how, notes=excluded.notes`
   - After both branches: await conn.commit(); SELECT back by id; return ReaderInformationState

5. `log_dramatic_irony(chapter_id: int, reader_knows: str, character_doesnt_know: str, character_id: int | None = None, irony_type: str = "situational", tension_level: int = 5, resolved_chapter_id: int | None = None, notes: str | None = None) -> DramaticIronyEntry | GateViolation`
   - Append-only INSERT (no ON CONFLICT) into dramatic_irony_inventory
   - Columns: chapter_id, reader_knows, character_id, character_doesnt_know, irony_type, tension_level, resolved_chapter_id, notes
   - resolved defaults to FALSE (not in INSERT columns — let DB default handle it)
   - Use cursor.lastrowid; SELECT back by id; return DramaticIronyEntry
