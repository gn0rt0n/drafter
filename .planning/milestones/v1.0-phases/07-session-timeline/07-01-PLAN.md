Implement these 7 tools in order:

**start_session() -> SessionStartResult | GateViolation**
Return type is now `SessionStartResult`, not `SessionLog` (per user decision in CONTEXT.md: "briefing content returned by start_session"). Logic:
1. Find open session: `SELECT * FROM session_logs WHERE closed_at IS NULL ORDER BY started_at DESC LIMIT 1`. If found, auto-close: `UPDATE session_logs SET closed_at = datetime('now'), notes = 'auto-closed by new session start' WHERE id = ?`. Commit.
2. Get briefing: `SELECT * FROM session_logs WHERE closed_at IS NOT NULL ORDER BY started_at DESC LIMIT 1`. Store as `prior_session` (None if no rows).
3. INSERT new session: `INSERT INTO session_logs DEFAULT VALUES`. Get `cursor.lastrowid`. Commit.
4. SELECT back new row by id. Build `new_session = SessionLog(**dict(row))`.
5. Build `briefing = SessionLog(**dict(prior_row))` if prior_session row exists, else `None`.
6. Return `SessionStartResult(session=new_session, briefing=briefing)`.

Do NOT log the briefing to stderr. The entire point is that briefing travels in the tool response body so Claude can read it as an MCP caller.

**close_session(session_id: int, summary: str | None = None, word_count_delta: int = 0, chapters_touched: list[int] | None = None) -> SessionLog | NotFoundResponse | GateViolation**
1. Verify session exists and is open: `SELECT * FROM session_logs WHERE id = ? AND closed_at IS NULL`.
2. If not found: return `NotFoundResponse(not_found_message=f"Open session {session_id} not found")`.
3. Auto-populate carried_forward: `SELECT question FROM open_questions WHERE answered_at IS NULL ORDER BY created_at ASC`. Build list of question strings.
4. UPDATE: `UPDATE session_logs SET closed_at = datetime('now'), summary = COALESCE(?, summary), word_count_delta = ?, chapters_touched = ?, carried_forward = ? WHERE id = ?` with (summary, word_count_delta, json.dumps(chapters_touched) if chapters_touched else None, json.dumps(carried), session_id).
5. Commit. SELECT back updated row. Return `SessionLog(**dict(updated[0]))`.

**get_last_session() -> SessionLog | NotFoundResponse | GateViolation**
`SELECT * FROM session_logs ORDER BY started_at DESC LIMIT 1`. If empty: return `NotFoundResponse(not_found_message="No session logs found")`. Return `SessionLog(**dict(rows[0]))`.

**log_agent_run(agent_name: str, tool_name: str, input_summary: str | None = None, output_summary: str | None = None, duration_ms: int | None = None, success: bool = True, session_id: int | None = None, error_message: str | None = None) -> AgentRunLog | GateViolation**
Append-only INSERT (no ON CONFLICT — audit trail). `INSERT INTO agent_run_log (session_id, agent_name, tool_name, input_summary, output_summary, duration_ms, success, error_message) VALUES (?, ?, ?, ?, ?, ?, ?, ?)`. Get lastrowid. Commit. SELECT back and return `AgentRunLog(**dict(rows[0]))`.

**get_project_metrics() -> ProjectMetricsSnapshot | GateViolation**
Live aggregate (does NOT read from project_metrics_snapshots — locked decision):
- `SELECT COALESCE(SUM(word_count), 0) AS word_count FROM chapters`
- `SELECT COUNT(*) AS cnt FROM chapters`
- `SELECT COUNT(*) AS cnt FROM scenes`
- `SELECT COUNT(*) AS cnt FROM characters`

Return `ProjectMetricsSnapshot(word_count=..., chapter_count=..., scene_count=..., character_count=..., session_count=...)` — omit `id` and `snapshot_at` (they have defaults or are None for live results).

**log_project_snapshot(notes: str | None = None) -> ProjectMetricsSnapshot | GateViolation**
1. Run same 5 live aggregate queries as get_project_metrics to get current values.
2. `INSERT INTO project_metrics_snapshots (word_count, chapter_count, scene_count, character_count, session_count, notes) VALUES (?, ?, ?, ?, ?, ?)`. Get lastrowid. Commit.
3. SELECT back by id. Return `ProjectMetricsSnapshot(**dict(rows[0]))`.
