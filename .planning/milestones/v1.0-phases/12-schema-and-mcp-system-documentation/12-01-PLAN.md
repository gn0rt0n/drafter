    Create the docs/ directory (it does not yet exist) and write docs/README.md as the entry point for a system-admin-level reader.

    Before writing, read these source files to verify actual entry points and structure:
    - pyproject.toml (verify `novel` and `novel-mcp` entry point definitions)
    - src/novel/mcp/server.py (verify server structure and how tools are registered)
    - src/novel/cli.py (verify CLI subcommand structure)

    docs/README.md must cover:

    1. **Title + one-paragraph overview** — what drafter is: a Python MCP server + SQLite narrative database + UV-managed CLI for AI-assisted novel writing. Written for a system admin who needs to understand implementation, not just usage.

    2. **Three-layer architecture section** — describe the three layers and how they connect:
       - Layer 1: SQLite database (22 migrations, 71 tables) — the persistence layer
       - Layer 2: MCP server (103 tools across 18 modules via FastMCP) — the AI-callable API layer
       - Layer 3: CLI (`novel` command via Typer) — the human-operated management layer
       Include a simple text diagram or Mermaid flowchart showing Claude Code -> MCP server -> SQLite and Human -> CLI -> SQLite.

    3. **Entry points section** — how to run each layer:
       - `uv run novel-mcp` — starts MCP server (stdio transport, consumed by Claude Code)
       - `uv run novel [subcommand]` — CLI for database management, seeding, gate operations, queries

    4. **Gate system section** — brief explanation of the architecture gate (36-item checklist that must be certified before prose-phase MCP tools will execute). This is a critical architectural concept that a system admin must understand.

    5. **Error contract section** — the three response types every MCP tool can return:
       - null with `not_found_message` for missing records
       - `is_valid: false` with `errors` for validation failures
       - `requires_action` for gate violations
       Document that no tool ever raises an exception.

    6. **Documentation map section** — where to find what:
       - `docs/schema.md` — full schema reference (71 tables, domain-organized, Mermaid ER diagrams)
       - `docs/mcp-tools.md` — full MCP tool reference (103 tools, 18 domains, tool cards with parameters/returns/gate status)
       - `project-research/database-schema.md` — historical design document (NOTE: may have drifted from actual implementation; see docs/schema.md for accurate field names)

    7. **Tech stack section** — brief: Python 3.12+, Pydantic v2, aiosqlite (MCP), sqlite3 (CLI), FastMCP (mcp>=1.26.0), Typer, uv/Hatchling.

    Target length: 80-150 lines. Tone: technical, precise. No marketing language.
