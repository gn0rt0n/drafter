Gate check implementation (from src/novel/mcp/gate.py):
  check_gate() queries architecture_gate table, returns GateViolation if is_certified=0, None if certified
  All gated tools follow: async with get_connection() as conn: violation = await check_gate(conn); if violation: return violation

Known implementation notes (verified, use these to write accurate tool docs):
  - Characters domain: 4 separate state tools exist (get_character_location, get_character_injuries, get_character_beliefs, get_character_knowledge) — NOT a single get_character_state
  - Relationships: get_relationship queries both orderings (min/max canonicalization); upsert_relationship also canonicalizes
  - Names: gate-free by design — name tools must work during worldbuilding phase before gate is certified
  - Session: start_session auto-closes open sessions before inserting new one
  - Session: close_session auto-populates carried_forward from unanswered open_questions
  - Canon tools (canon_facts, decisions_log, continuity_issues): append-only INSERT (no ON CONFLICT) — audit log tables
  - Foreshadowing: log_foreshadowing is upsert (allows payoff to be filled in later); log_motif_occurrence is append-only
  - Gate tools: run_gate_audit and certify_gate are independent — audit does not certify
