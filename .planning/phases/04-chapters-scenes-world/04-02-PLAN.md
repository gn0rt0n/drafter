DO NOT use INSERT OR REPLACE on tables with FK children — use ON CONFLICT DO UPDATE (Phase 3 decision).

    - Signature: `async def get_scene_character_goals(scene_id: int) -> list[SceneCharacterGoal] | NotFoundResponse`
    - First verify scene exists: `SELECT id FROM scenes WHERE id = ?`; return `NotFoundResponse` if miss
    - Then: `SELECT * FROM scene_character_goals WHERE scene_id = ? ORDER BY id`
    - Return `[SceneCharacterGoal(**dict(row)) for row in rows]` (empty list is valid if no goals)

    **Tool 3 — upsert_scene (CHAP-08):**
    - Signature: `async def upsert_scene(scene_id: int | None, chapter_id: int, scene_number: int, location_id: int | None = None, time_marker: str | None = None, summary: str | None = None, scene_type: str = "action", dramatic_question: str | None = None, scene_goal: str | None = None, obstacle: str | None = None, turn: str | None = None, consequence: str | None = None, emotional_function: str | None = None, narrative_functions: list[str] | None = None, word_count_target: int | None = None, status: str = "planned", notes: str | None = None) -> Scene | ValidationFailure`
    - Two-branch pattern:
      - `scene_id is None`: INSERT without id. Build a `Scene(chapter_id=chapter_id, scene_number=scene_number, ...)` object, call `scene.to_db_dict()` to get JSON-serialized dict. Use the dict values for the INSERT. `cursor.lastrowid` for new id.
      - `scene_id is not None`: `INSERT INTO scenes (id, chapter_id, scene_number, ..., narrative_functions, ...) VALUES (?, ?, ...) ON CONFLICT(id) DO UPDATE SET chapter_id=excluded.chapter_id, ..., updated_at=datetime('now')`. Pass `json.dumps(narrative_functions)` for the narrative_functions parameter (or use to_db_dict() approach).
    - After write: SELECT back by id to return the canonical Scene row
    - Wrap in try/except; return `ValidationFailure(is_valid=False, errors=[str(exc)])` on error

    **Tool 4 — upsert_scene_goal (CHAP-09):**
    - Signature: `async def upsert_scene_goal(scene_id: int, character_id: int, goal: str, obstacle: str | None = None, outcome: str | None = None, notes: str | None = None) -> SceneCharacterGoal | ValidationFailure`

    - After write: `SELECT * FROM scene_character_goals WHERE scene_id = ? AND character_id = ?`
    - Return `SceneCharacterGoal(**dict(row[0]))`
    - Wrap in try/except; return `ValidationFailure(is_valid=False, errors=[str(exc)])` on error

    **Module-level rules:**

    - `import json` (for narrative_functions serialization in upsert_scene)
    - Return error types — NEVER raise
