async def _call_tool(db_path: str, tool_name: str, args: dict):
    os.environ["NOVEL_DB_PATH"] = db_path
    async with create_connected_server_and_client_session(mcp) as session:

FastMCP serialization (list vs single):
- list[T] tools: `[json.loads(c.text) for c in result.content]`
- Single-object tools: `json.loads(result.content[0].text)`
- Empty list: result.content is empty — `[json.loads(c.text) for c in result.content]` returns []

Tool signatures and behavior (8 tools):

1. `get_foreshadowing(status: str | None = None, chapter_id: int | None = None) -> list[ForeshadowingEntry] | GateViolation`
   - Build WHERE: if status add `status = ?`; if chapter_id add `plant_chapter_id = ?`
   - SQL: `SELECT * FROM foreshadowing_registry` + WHERE + `ORDER BY plant_chapter_id ASC`

2. `get_prophecies() -> list[ProphecyEntry] | GateViolation`
   - SQL: `SELECT * FROM prophecy_registry ORDER BY created_at ASC`

3. `get_motifs() -> list[MotifEntry] | GateViolation`
   - SQL: `SELECT * FROM motif_registry ORDER BY created_at ASC`

4. `get_motif_occurrences(motif_id: int | None = None, chapter_id: int | None = None) -> list[MotifOccurrence] | GateViolation`
   - Build WHERE: if motif_id add `motif_id = ?`; if chapter_id add `chapter_id = ?`
   - SQL: `SELECT * FROM motif_occurrences` + WHERE + `ORDER BY created_at ASC`

5. `get_thematic_mirrors() -> list[ThematicMirror] | GateViolation`
   - SQL: `SELECT * FROM thematic_mirrors ORDER BY created_at ASC`

6. `get_opposition_pairs() -> list[OppositionPair] | GateViolation`
   - SQL: `SELECT * FROM opposition_pairs ORDER BY created_at ASC`

7. `log_foreshadowing(description: str, plant_chapter_id: int, foreshadowing_id: int | None = None, plant_scene_id: int | None = None, payoff_chapter_id: int | None = None, payoff_scene_id: int | None = None, foreshadowing_type: str = "direct", status: str = "planted", notes: str | None = None) -> ForeshadowingEntry | GateViolation`
   - LOCKED DECISION: two-branch upsert (NOT append-only)
   - None-id branch: plain INSERT into foreshadowing_registry (description, plant_chapter_id, plant_scene_id, payoff_chapter_id, payoff_scene_id, foreshadowing_type, status, notes); use cursor.lastrowid
   - Provided-id branch: INSERT with ON CONFLICT(id) DO UPDATE SET description=excluded.description, plant_chapter_id=excluded.plant_chapter_id, plant_scene_id=excluded.plant_scene_id, payoff_chapter_id=excluded.payoff_chapter_id, payoff_scene_id=excluded.payoff_scene_id, foreshadowing_type=excluded.foreshadowing_type, status=excluded.status, notes=excluded.notes, updated_at=datetime('now'); new_id = foreshadowing_id
   - commit(); SELECT back by id; return ForeshadowingEntry

8. `log_motif_occurrence(motif_id: int, chapter_id: int, scene_id: int | None = None, description: str | None = None, occurrence_type: str = "direct", notes: str | None = None) -> MotifOccurrence | GateViolation`
   - LOCKED DECISION: append-only INSERT (historical events — NOT upsert)
   - Append-only INSERT into motif_occurrences (motif_id, chapter_id, scene_id, description, occurrence_type, notes)
   - cursor.lastrowid; commit(); SELECT back by id; return MotifOccurrence

Critical rules: every tool calls check_gate(conn) first; no print(); await conn.commit() after every write.
